feat(trip-list): 新增行程列表页面与完善详情页交互逻辑

行程列表页面（TripList.vue）：
- 创新卡片瀑布流布局，自适应网格（28vw 最小宽度）
- 精美的空状态设计：浮动飞机图标 + 三层扩散波纹动画
- 卡片展示：标题、目的地、日期、天数、人数、预算（绿色渐变高亮）
- 悬停动画：放大 + 上移 + 增强阴影，箭头右移
- 加载状态：旋转 spinner 动画
- 点击卡片跳转详情页，携带 fromPage: 'list' 标记
- 右上角渐变装饰元素，底部创建时间显示

行程详情页交互优化：
- 智能返回逻辑：
  * 从列表进入 → 返回列表页（/trips）
  * 从创建页进入 → 返回创建页并保留用户输入
  * 其他情况 → 返回首页
- 从 API 加载行程数据（GET /trips/{tripId}/itinerary）
- 按钮显示逻辑优化：
  * 使用 isNewTrip 标记（默认 false，避免闪烁）
  * 仅未确认行程（!updatedAt）显示确认/重新生成按钮
  * 确认后按钮消失

重新生成功能：
- 点击直接调用 POST /trips 接口，无需返回创建页
- 复用 CreateTrip 的进度弹窗样式（Vue 模板 + transition）
- 地球旋转动画 + 渐变进度条 + 动态提示文字
- 防止重复点击（isRegenerating 状态锁）
- 生成完成后自动跳转到新行程详情页

路由与导航：
- 新增 /trips 路由（TripList 组件）
- Home 页面"查看行程"按钮跳转到列表页
- CreateTrip 跳转详情时携带 fromPage: 'create' 和 userInput

后端适配（TripService.java）：
- 修复行程生成接口的数据返回格式

UI/UX 细节：
- Toast 通知替代原生 alert
- 进度条高度统一为 0.6vw（与 CreateTrip 一致）
- 按钮过渡动画流畅
- 日期格式化（YYYY-MM-DD）
- 行程天数自动计算（包含首尾）
